## Overview
This is the BLE (Bluetooth Low Energy) GATT server that runs on the Raspberry Pi 4. It turns the RP4 into a BLE Peripheral that advertises a custom service, accepts motor control commands from the PARMCO phone app, and sends RPM telemetry back to the phone.

For Checkpoint 2, the server accepts commands and logs them to the console. The telemetry characteristic sends a mock RPM value derived from the current speed setting. Real RPM sensor integration comes in CP3.

## Architecture

```
Phone (Central)                     RP4 (Peripheral)
┌─────────────────┐                ┌──────────────────────────────┐
│  PARMCO App      │   BLE Write   │  ble_server                  │
│  (React Native)  │ ────────────> │  Command Char (Write)        │
│                  │               │    → parse_command()          │
│  RPM Display  <──│── BLE Notify  │  Telemetry Char (Notify)     │
│                  │               │    → PropertiesChanged signal │
└─────────────────┘                └──────────────────────────────┘
```

### GATT Service Structure

| Element | UUID | Properties |
|---------|------|------------|
| **Service** | `12345678-1234-5678-1234-56789abcdef0` | Primary |
| **Command** | `12345678-1234-5678-1234-56789abcdef1` | Write |
| **Telemetry** | `12345678-1234-5678-1234-56789abcdef2` | Read, Notify |

### Command Protocol

Commands are sent as UTF-8 strings from the phone:

| Command | Action |
|---------|--------|
| `FWD` | Set direction to forward (clockwise) |
| `REV` | Set direction to reverse (counterclockwise) |
| `START` | Turn motor on |
| `STOP` | Turn motor off |
| `SPD:XX` | Set speed to XX percent (e.g., `SPD:75`) |

## Prerequisites

- Raspberry Pi 4 Model B
- Raspberry Pi OS (64-bit, Debian Bookworm recommended)
- Bluetooth hardware enabled (built into the RP4)

## Setup

### 1. Install dependencies
```bash
sudo apt update
sudo apt install -y build-essential libglib2.0-dev bluetooth bluez libbluetooth-dev pkg-config
```

### 2. Verify BlueZ is running
```bash
# Check that the Bluetooth service is active
sudo systemctl status bluetooth

# If not running:
sudo systemctl enable bluetooth
sudo systemctl start bluetooth

# Verify the adapter is visible
sudo bluetoothctl
# Inside bluetoothctl:
#   power on
#   discoverable on
#   exit
```

### 3. Compile
```bash
make
```

This produces the `ble_server` binary (compiled from `ble_server.c`).

### 4. Run manually (for testing)
```bash
sudo ./ble_server
```

You should see:

### 5. Install as a boot service (for demo)

This copies the binary to `/usr/local/bin/`, installs the systemd service, and enables auto-start. The RP4 will start the BLE server on every boot.

```bash

```

## Testing with the Phone App

1. Start the BLE server on the RP4 (`sudo ./ble_server` or via systemd).
2. Open the PARMCO app on the phone.
3. Tap "Connect" — the app scans for the PARMCO service UUID.
4. The app should discover "PARMCO-RP4" and connect.
5. Use the UI controls — commands will appear in the RP4 terminal.
6. To test "Forget and Reconnect":
   - Go to iPhone Settings → Bluetooth → find PARMCO-RP4 → "Forget This Device"
   - In the PARMCO app, tap "Connect" again
   - The connection should re-establish successfully

## Troubleshooting

### "Failed to register GATT application"

- Make sure BlueZ is running: `sudo systemctl status bluetooth`
- Make sure no other GATT application is registered on the same adapter
- Try restarting BlueZ: `sudo systemctl restart bluetooth`
- Check BlueZ version: `bluetoothctl --version` (needs 5.50+)

### "Cannot connect to system D-Bus"

- The program must run as root: `sudo ./ble_server`
- Or if using systemd, the service runs as root by default

### Phone cannot discover the RP4

- Verify the adapter is powered: `sudo bluetoothctl` → `power on`
- Check that advertising is registered: look for the "BLE advertisement registered" message in the server output
- Make sure the phone's Bluetooth is on and the app has BLE permissions

### Phone connects but commands don't arrive

- Check the phone app's Service UUID matches exactly: `12345678-1234-5678-1234-56789abcdef0`
- Check the Command Characteristic UUID: `12345678-1234-5678-1234-56789abcdef1`
- Verify the command format (UTF-8 strings: "FWD", "REV", "START", "STOP", "SPD:XX")

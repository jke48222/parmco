CP 01 FEB 2026

Design & planning.

The opening checkpoint established project scope and produced initial hardware documentation. The team defined the system architecture: phone app communicating over BLE to a Pi-based motor controller. Research covered the L293D H-bridge, the BCM2711 GPIO peripheral model, BlueZ GATT server programming, and the Nichibo PC-260SB motor characteristics.

An AI-generated schematic and technical documentation were produced alongside a human-written strategy document describing the implementation plan, risk areas, and division of labour.

CP 02 MAR 2026

BLE, iOS & keyboard motor control.

Three working subsystems, built independently before integration.

BLE GATT server

ble_server.c — a C program using GLib / GDBus to register a custom GATT service with BlueZ, advertise as PARMCO-RP4, accept motor commands, and return mock RPM telemetry (speed% × 100). No motor output connected yet.

iOS application

Native SwiftUI + CoreBluetooth, with a parallel React Native + Expo build. Both scan for the PARMCO service UUID, connect, send ASCII command strings, and display incoming RPM notifications. SwiftUI is used for all demos.

Motor keyboard control

motor_control.c paired with an ARMv7 assembly GPIO module. Non-blocking terminal input via termios + select(). pigpio DMA-based 20 kHz PWM on GPIO18. Three control modes already present: Manual, Maintain, and Match. BLE and motor subsystems ran independently.

CP 03 APR 2026

Integration, IR feedback, autostart.

The third checkpoint connected all CP2 subsystems into a single end-to-end system and added the hardware features required for closed-loop control.

BLE integration

ble_server.c now calls motor-control functions directly on every received command. A phone tap on Start sends START over BLE; parse_command() sets motor_state.running = 1; gpioPWM() drives the enable pin immediately.

Real RPM telemetry

An IR sensor on the propeller is wired to GPIO17. pigpio's alert callback counts rising edges with an 800 µs debounce. The telemetry characteristic now emits RPM:<int> at 5 Hz while running, 1 Hz at idle.

Pairing agent

A NoInputNoOutput BlueZ Agent1 at /com/parmco/agent auto-accepts all iOS pairing requests. Without this agent, the connection hangs indefinitely on "waiting for response from Bluetooth device."

systemd autostart

parmco-ble.service starts ble_server after bluetooth.service on every boot, with Restart=on-failure. The CP3 check sheet tests autonomous reconnect after power cycle and "Forget This Device" — both pass without any console input on the Pi.

FINAL APR 2026

All three control modes, on one tap.

All three modes in one build. We walk through ten things on stage — cold boot, scan, connect, forward, reverse, start, stop, speed ramp, target RPM, and a forget-device + power-cycle reconnect — and nobody touches the Pi's keyboard.

Manual

Direct PWM duty from the phone via SPD:XX. No feedback loop; percentage maps linearly to duty cycle.

Maintain

Proportional controller (Kp = 0.04) holds a target RPM sent via RPM:<value>, using IR feedback. ±30 RPM deadband prevents hunting at steady state.

Match

Motor tracks an external reference signal on GPIO23 using the same proportional controller against live IR feedback.

§ SOURCE CODE

The final implementation.

Two excerpts carry the weight of the whole project: the proportional controller that closes the loop, and the ARM assembly that writes the GPIO registers directly.

final_project/motor_control_cp3.c C · P-controller 
/* Proportional controller — runs every 250 ms under the maintain mode */
static void run_controller(void)
{
    int error = target_rpm - current_rpm;
    if (abs(error) < DEADBAND_RPM) return;   // ±30 RPM — don't hunt

    float correction = KP * error;                 // Kp = 0.04
    int   new_duty   = current_duty + (int)correction;

    if (new_duty < 0)   new_duty = 0;
    if (new_duty > 255) new_duty = 255;

    gpioPWM(PIN_ENABLE, new_duty);
    current_duty = new_duty;
}
final_project/gpio_asm.s ARMv7 · L190 
// gpio_write(pin, value) — atomic GPIO write via BCM2711 registers
.global gpio_write
.type    gpio_write, %function

gpio_write:
    ldr     r2, =gpio_base_addr   // R2 ← &base
    ldr     r2, [r2]              // R2 ← base
    mov     r3, #1
    lsl     r3, r3, r0            // R3 ← 1 << pin
    cmp     r1, #0
    beq     .Lclear_pin
    str     r3, [r2, #0x1C]       // GPSET0 → pin HIGH
    bx      lr
.Lclear_pin:
    str     r3, [r2, #0x28]       // GPCLR0 → pin LOW
    bx      lr
